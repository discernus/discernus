    1. Schema Generator (src/cli/schema_generator.py)

    Purpose: Generate JSON Schema skeletons from example records and refine with descriptions and required flags.

    Key Features: - ‚úÖ Automatic type detection (strings, numbers, objects, arrays) - ‚úÖ Format detection (date-time, URI, email patterns) - ‚úÖ Enum value detection for limited value sets - ‚úÖ Smart required field detection (‚â•80% presence threshold) - ‚úÖ Recursive analysis of nested objects and arrays - ‚úÖ Schema validation against existing schemas - ‚úÖ Comprehensive CLI with examples and help

    2. JSONL Generator (src/cli/jsonl_generator.py)

    Purpose: Convert various source formats to JSONL corpus files with multiple chunking strategies.

    Supported Input Formats: - ‚úÖ Markdown (.md, .markdown) with YAML frontmatter support - ‚úÖ CSV files with configurable text and metadata columns - ‚úÖ Plain text files with command-line metadata override

    Chunking Strategies: - ‚úÖ Fixed chunking: Configurable size with overlap, word-boundary aware - ‚úÖ Sectional chunking: Splits by headers, paragraphs, numbered sections - ‚úÖ Semantic chunking: Sentence/paragraph boundary aware with size limits

    Metadata Processing: - ‚úÖ YAML frontmatter parsing for Markdown files - ‚úÖ CSV column mapping for structured data - ‚úÖ JSON metadata override for plain text - ‚úÖ Automatic metadata normalization and validation

    Output Features: - ‚úÖ Core schema compliance validation - ‚úÖ Automatic chunk metadata calculation (word counts, density, positioning) - ‚úÖ Framework extension data support - ‚úÖ Comprehensive error reporting and validation
